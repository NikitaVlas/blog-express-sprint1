Установка зависимостей

pnpm add express

pnpm add -D typescript @types/express jest ts-jest @types/jest @types/node supertest @types/supertest eslint @typescript-eslint/eslint-plugin @typescript-eslint/parser prettier eslint-config-prettier eslint-plugin-prettier nodemon

typescript - язык программирования, являющийся надмножеством JavaScript, добавляющий статическую типизацию. Мы используем TypeScript для написания более безопасного и масштабируемого кода с использованием типов.

@types/express - типы для библиотеки Express, написанные для TypeScript. TypeScript требует типы для всех библиотек, с которыми мы работаем. Этот пакет предоставляет типы для Express, чтобы мы могли работать с Express в TypeScript без ошибок компиляции.

jest - тестовый фреймворк для JavaScript, который поддерживает тестирование кода. Используется для написания и выполнения юнит-тестов, тестов интеграции и моков.

ts-jest - преобразователь для Jest, позволяющий тестировать TypeScript-код. Позволяет Jest работать с TypeScript, компилируя его во время тестирования.

@types/jest - типы для jest.

@types/node - типы для Node.js. Позволяет использовать Node.js API с TypeScript, обеспечивая поддержку типов для встроенных модулей Node.js.

supertest - библиотека для e2e тестирования HTTP-запросов. Используется для тестирования API, отправляя запросы к серверу и проверяя ответы.

@types/supertest - Типы для библиотеки supertest.

eslint - линтер для JavaScript и TypeScript. Используется для анализа кода на наличие ошибок и проблем с качеством, таких как несоответствие стилю кодирования или потенциальные ошибки.

@typescript-eslint/eslint-plugin - плагин для ESLint, который добавляет поддержку TypeScript. Расширяет возможности ESLint, добавляя правила для проверки TypeScript-кода.

@typescript-eslint/parser - парсер для ESLint, который позволяет ESLint анализировать TypeScript-код. Обеспечивает совместимость ESLint с TypeScript.

prettier - форматировщик кода. Используется для автоматического форматирования кода по установленным правилам, улучшая читаемость и единообразие.

eslint-config-prettier - конфигурация для ESLint, которая отключает правила, которые могут конфликтовать с Prettier. Помогает интегрировать Prettier с ESLint без конфликтов, отключая проверки, которые дублируют функциональность Prettier.

eslint-plugin-prettier - плагин для ESLint, который интегрирует Prettier в процесс линтинга. Обеспечивает использование Prettier как ESLint-правила, что позволяет линтить код и форматировать его в одном процессе.

nodemon - утилита для автоматической перезагрузки сервера при изменениях в исходных файлах. Упрощает разработку, автоматически перезапуская сервер при изменении файлов, что позволяет ускорить процесс разработки.


Инициализация TypeScript

pnpm tsc --init

Файл tsconfig.json содержит основные настройки компиляции кода из TypeScript в JavaScript. Здесь указываются:

формат выходного кода (target, module — например, CommonJS),

структура проекта (где исходный код и куда сохраняется скомпилированный — rootDir, outDir),

включаемые/исключаемые файлы (include, exclude),

а также базовые опции, улучшающие совместимость и строгую проверку типов.


Настройка скриптов в package.json
"scripts": {
  "watch": "tsc -w",
  "dev": "nodemon --inspect dist/index.js",
  "jest": "jest -i",
  "lint": "eslint --fix .",
  "format": "prettier --write ."
}

watch: компилирует файлы TypeScript из папки src в dist в режиме наблюдения (-w).

dev: запускает приложение с помощью nodemon, следя за изменениями в скомпилированном коде в dist.

jest: запускает тесты с помощью Jest.

lint: запускает проверку кода с помощью ESLint и автоматически исправляет ошибки.

format: форматирует код с помощью Prettier.


Запуск компиляции TypeScript

Запускаем команду для компиляции TypeScript в режиме наблюдения:

pnpm watch


Запуск приложения

Запускаем приложение с помощью nodemon, чтобы следить за изменениями в коде:

pnpm dev
